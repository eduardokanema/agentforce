# Verification Catalog

V1 keeps verification lightweight and operator-readable.

## Evidence Classes

- contract or schema validation
- command or artifact proof
- evaluator or reviewer result
- stream or event summary
- changed artifact summary

## Initial Mapping

- draft validation -> contract summary
- planning critics and resolver -> evaluator summary
- task artifacts and outputs -> artifact summary
- mission and task stream events -> stream summary
- review feedback and score -> verifier summary

## V1 Rule

Every project cycle should expose enough evidence for an operator to validate the current next step without opening raw logs first.
